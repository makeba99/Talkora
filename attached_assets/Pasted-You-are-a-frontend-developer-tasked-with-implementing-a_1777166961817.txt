You are a frontend developer tasked with implementing a new animated decorations system for user profiles. Your goal is to create a polished, professional-grade feature that feels cohesive with the platform's existing design language and allows users to add animated decorative elements to their profiles—similar to how Discord implements profile decorations.

**Core Requirements:**

- Design and implement animated decoration components that feel **serious and professional**, not playful or casual
- Ensure decorations integrate seamlessly with the existing platform design system and visual hierarchy
- Create a system where users can easily add, preview, and manage decorations on their profiles
- Decorations should display consistently and maintain visual polish across different profile layouts and screen sizes
- Include smooth animations that enhance rather than distract from the profile itself

**Technical Considerations:**

- Use the platform's existing component library and styling conventions
- Optimize animations for performance (consider GPU acceleration, frame rate efficiency)
- Ensure animations respect user accessibility preferences (prefers-reduced-motion)
- Build the feature to scale—the system should support adding new decorations without architectural changes
- Handle edge cases: overlapping decorations, mobile responsiveness, loading states

**Design Direction:**

- Study how similar platforms (Discord, Twitch, other professional social platforms) implement profile decorations
- The aesthetic should feel premium and intentional, not cluttered
- Animations should be subtle and sophisticated—avoid anything that feels gimmicky
- Consider how decorations coexist with existing profile elements (bio, avatar, badges, etc.)

**Deliverables:**

- Fully functional animated decoration components ready for integration
- A decoration picker/management UI for users to select and customize their decorations
- Profile preview showing how decorations will appear in context
- Proper error handling and loading states
- Code follows platform conventions and is production-ready

Start by outlining the component structure, then implement the feature end-to-end.
