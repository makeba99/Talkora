ROLE: Senior AI Systems Engineer

TASK:
Rebuild the current AI system to use NVIDIA Nemotron Nano as the primary model and implement a real-time voice interaction pipeline. Remove all unused or legacy logic from the previous GPT-4 implementation.

GOAL:
- Instant, real-time voice interaction
- No repeated or generic responses
- Clean, efficient architecture
- Smooth YouTube/media coexistence (no conflicts)

---

CORE ARCHITECTURE:

Implement a real-time voice pipeline:

User Voice →
Speech-to-Text (STT) →
Nemotron Nano (LLM, streaming) →
Text chunks →
Text-to-Speech (TTS, streaming) →
Audio output (plays immediately)

---

1. REMOVE LEGACY GPT-4 LOGIC

- Remove:
  - GPT-4 default routing
  - Blocking (non-streaming) responses
  - Redundant prompt wrappers
  - Old retry loops tied to GPT behavior
- Keep ONLY optional fallback hook (disabled by default)

---

2. NVIDIA NEMOTRON INTEGRATION

- Model: nemotron-3-nano-30b
- Endpoint: NVIDIA NIM API
- Enable: stream = true

Requirements:
- Start receiving tokens immediately
- Do NOT wait for full completion
- Pass minimal, clean prompt (no heavy system overload)

---

3. REAL-TIME STREAMING ENGINE (CRITICAL)

- Stream tokens from Nemotron
- Buffer into small chunks (e.g., sentence or phrase level)
- Send chunks instantly to TTS

DO NOT:
- Wait for full response
- Block UI while generating

TARGET:
- First token < 300ms
- Voice start < 700ms

---

4. TEXT-TO-SPEECH (TTS)

- Integrate fast TTS engine (prefer NVIDIA Riva or equivalent)
- Support:
  - Low latency
  - Natural voice
  - Streaming playback

Behavior:
- Speak as chunks arrive
- Interrupt previous speech if user speaks again

---

5. SPEECH-TO-TEXT (STT)

- Real-time transcription
- Detect end-of-speech quickly
- Avoid long silence delays

---

6. INTERRUPT + LISTENING LOGIC

- If user speaks while AI is talking:
  → Immediately stop TTS
  → Cancel current AI generation
  → Process new input
