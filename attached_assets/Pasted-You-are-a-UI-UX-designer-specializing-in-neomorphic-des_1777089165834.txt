You are a UI/UX designer specializing in neomorphic design systems. Your task is to redesign the default theme of an application to match a comprehensive neomorphic aesthetic, with all UI components, icons, and interactive elements updated to follow this visual language.

**Reference Material:**
You have two neomorphic design references (image.png files) that showcase the complete design system you should match:
- One features a dark deep purple/blue color palette with magenta-to-cyan gradient accents for interactive states
- One features a light off-white/grey palette with soft purple accents for active states

Both demonstrate the core neomorphic principles: soft, diffused lighting that creates embossed (raised) and debossed (inset) effects, smooth rounded corners, subtle shadows and highlights, and a cohesive tactile 3D appearance.

**Design Requirements:**

1. **Color Palette & Lighting:** Match the base color palette from the reference images (either dark or light, as appropriate for your application). Implement consistent, diffused shadows and highlights across all components to create the sculpted neomorphic depth.

2. **All UI Components:** Redesign every component to follow neomorphic principles:
   - Buttons (circular, square, pill-shaped) with appropriate raised or recessed states
   - Input fields and search bars with inset/debossed appearance
   - Toggles, sliders, and progress bars with soft, tactile styling
   - Card containers and modal backgrounds with seamless integration
   - Chat bubbles and message containers

3. **Icons:** Redesign or restyle all icons to match the neomorphic system—simple, clean lines (outlined or filled) with consistent sizing and alignment. Active icons should feature gradient fills or soft highlights matching the reference palette (magenta-to-cyan for dark theme, purple accents for light theme).

4. **Interactive States:** Define clear active, hover, and disabled states for all elements using subtle lighting shifts, gradient fills, or depth changes—never rely on color alone.

5. **Consistency:** Ensure all components share the same visual language, shadow depth, corner radius treatment, and overall aesthetic so the theme feels unified and cohesive.

**Deliverable:**
Provide a complete redesigned theme specification or component library (in your chosen format: design tokens, CSS/code, design system document, or visual mockups) that can be applied as the default theme across the entire application.
