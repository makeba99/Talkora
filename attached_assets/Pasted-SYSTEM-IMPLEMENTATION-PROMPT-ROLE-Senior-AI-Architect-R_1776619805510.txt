SYSTEM / IMPLEMENTATION PROMPT

ROLE:
Senior AI Architect & Real-Time Voice Avatar Engineer

MISSION:
Build a real-time AI avatar system powered by NVIDIA Nemotron-3 Nano 30B, capable of voice conversation + speech recognition + realistic talking face animation with ultra-low latency.

PRIMARY GOAL:
Create an instant-response AI tutor/avatar that listens, thinks, speaks, and animates a human-like face in real time.

---

🧠 1. AI BRAIN (CORE LLM)

MODEL:
Nemotron-3 Nano 30B (NVIDIA)

REQUIREMENTS:
- Must use streaming responses (token-by-token)
- No full-response waiting allowed
- Optimize for latency over complexity
- Maintain short, conversational outputs

RULES:
- Always respond in real-time streaming mode
- Avoid repetitive or generic answers
- Keep responses conversational and natural

---

🎤 2. SPEECH-TO-TEXT (STT MODULE)

MODEL OPTIONS:
- Whisper (OpenAI open-source)
- faster-whisper (RECOMMENDED for speed)

REQUIREMENTS:
- Real-time transcription (not batch)
- Detect end-of-speech automatically
- Low latency input processing (<300ms target)

RULES:
- Continuously listen when session is active
- Immediately send finalized text to AI brain

---

🗣️ 3. TEXT-TO-SPEECH (TTS MODULE)

OPTION A (RECOMMENDED - SPEED FIRST):
- Piper TTS (local, ultra-fast, fully free)

OPTION B (QUALITY MODE):
- Coqui TTS (more natural but slower)

REQUIREMENTS:
- Must support streaming audio output
- Must start speaking before full sentence completes
- Must support interruption (barge-in enabled)

RULES:
- Start speaking immediately when first chunk arrives
- Do NOT wait for full AI response

---

🎭 4. TALKING FACE / AVATAR SYSTEM

PRIMARY OPTION:
- LivePortrait (best free realistic face animation)

ALTERNATIVE:
- Wav2Lip (stable lip sync fallback)

REQUIREMENTS:
- Audio-driven facial animation
- Real-time or near-real-time lip sync
- Smooth expression transitions

RULES:
- Avatar animation must sync with TTS output
- No lag between speech and mouth movement
- Must support continuous streaming updates

---

⚡ 5. REAL-TIME PIPELINE (CRITICAL FLOW)

USER VOICE →
STT (Whisper / faster-whisper) →
Nemotron-3 Nano (streaming LLM) →
Chunked text output →
Piper TTS (streaming voice) →
LivePortrait (lip sync face animation)

---

🔁 6. STREAMING + INTERRUPT LOGIC

MANDATORY BEHAVIOR:
- Everything must be streaming (no full waits)
- AI must be interruptible at any time

IF user speaks:
→ immediately stop TTS
→ cancel AI generation
→ restart pipeline with new input

---

- Store last 5–10 messages only
- Avoid long context bloat
- Prevent repeated answers

RULE:

→ regenerate response once
