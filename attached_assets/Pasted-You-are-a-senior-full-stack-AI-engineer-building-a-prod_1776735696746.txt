You are a senior full-stack AI engineer building a production-grade real-time AI tutor avatar system. Your objective is to architect and implement a complete pipeline that replaces a 2D cartoon avatar with a photorealistic human face that speaks naturally, maintains perfect lip sync, and responds to voice input with minimal latency.

---

**TECHNICAL STACK (MANDATORY - NO SUBSTITUTIONS)**

AI Inference: NVIDIA Nemotron-3 Nano 30B with token streaming enabled
Speech Recognition: faster-whisper for real-time transcription
Text-to-Speech: Piper TTS (local, streaming-compatible)
Lip Sync Engine: MuseTalk (audio-driven mouth animation)
Facial Animation: LivePortrait (eye blinking, head motion, subtle expressions)

---

**CORE PIPELINE ARCHITECTURE**

Implement this exact sequence with no blocking operations:

User audio input → faster-whisper STT → Nemotron token stream → chunk-based TTS conversion → Piper audio stream → MuseTalk frame generation → LivePortrait expression layer → UI canvas render

Every stage must operate concurrently where possible. Do not wait for full AI completion before beginning TTS or animation.

---

**LATENCY TARGETS (NON-NEGOTIABLE)**

- First response token: < 500ms from end of user speech
- Audio playback start: < 700ms
- Lip sync latency: < 50ms relative to audio playback
- Frame rate: 30fps minimum for facial animation

---

**CRITICAL FUNCTIONAL REQUIREMENTS**

1. **Real-Time Streaming**: Nemotron must stream tokens incrementally. Convert each chunk to audio via Piper immediately—do not accumulate full responses. This is the primary driver for achieving sub-700ms voice latency.

2. **Synchronized Lip and Face Animation**: MuseTalk drives mouth movement precisely from audio stream. LivePortrait layers idle animations (breathing, blinking) and adds contextual expressions. Audio and visual elements must never drift; any sync drift > 50ms is a failure state.

3. **Human Face Source**: Use a real human face photograph or 2–5 second idle video loop as the avatar base. Static images without animation appear lifeless—inject subtle idle motion (breathing cycles, occasional blinks) even during silence to maintain presence.

4. **Interrupt and Context Switching**: If the user speaks while the AI is speaking, immediately halt TTS playback, cancel ongoing Nemotron generation, and restart the full pipeline. Do not queue or defer user input.

5. **Intent Extraction and Response Grounding**: Before generating responses, explicitly extract user intent from transcribed speech. Always reference the user's actual input in the response (e.g., "You asked about X, and here's why..."). Implement similarity checking—if a new response exceeds 80% lexical similarity to the previous response, force regeneration. This prevents generic or repetitive outputs.

6. **UI Integration**: Replace the current 2D avatar circle with a video canvas or WebGL render surface. Preserve existing controls (microphone toggle, AI on/off, speaking indicator). Add a waveform visualization synced to voice output and a subtle glow effect when the AI is speaking. The avatar system must integrate cleanly without disrupting existing UI layout.

7. **Media State Isolation**: Prevent the avatar render from interfering with other media on the page (YouTube players, audio elements). Maintain separate state containers: `aiState` (LLM/inference), `avatarState` (animation/rendering), `mediaState` (external media tracking). Use CSS containment and z-index management to prevent layer collisions.

8. **Performance Optimization**: Run MuseTalk and LivePortrait with GPU acceleration when available; implement frame batching to reduce re-render overhead. Profile for bottlenecks and optimize hot paths—avatar rendering should consume < 30% GPU when active.

9. **Graceful Degradation**: If the avatar pipeline fails (MuseTalk or LivePortrait errors), fall back to Wav2Lip or a simple animated talking head while maintaining voice response. Do not break the tutoring experience due to animation failures.

10. **Debug Mode**: Implement a hidden debug toggle ("AI Debug") that displays real-time metrics: end-to-end latency, active model names, streaming status, sync status (audio vs. lip position drift). Include frame timing and model inference time breakdowns. Disable in production but keep toggle accessible for troubleshooting.

Store API keys, model paths, and streaming parameters in `.env`. Make streaming chunk size, TTS buffer size, and animation frame rate configurable. Document all tunable latency parameters.

The system should feel like the user is on a live video call with a real human tutor: instantaneous response, natural speech, authentic facial presence, and genuine engagement. There should be no perceptible delay between the tutor's speech and mouth movement, no dead time, and no sense of mechanical processing.
