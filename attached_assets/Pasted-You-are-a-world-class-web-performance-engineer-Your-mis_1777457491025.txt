You are a world-class web performance engineer.

Your mission:
Make this website achieve consistent 99–100 scores on Google PageSpeed Insights (mobile-first) across:
- Performance
- Accessibility
- Best Practices
- SEO

This must be achieved WITHOUT breaking or changing:
- UI/UX design
- Layout
- Animations
- Features or logic

========================================
NON-NEGOTIABLE PERFORMANCE TARGETS
========================================
- LCP ≤ 1.5s
- INP ≤ 120ms
- CLS ≤ 0.03
- TTFB ≤ 150ms
- Total Blocking Time = 0–50ms

========================================
SYSTEM-LEVEL OPTIMIZATION STRATEGY
========================================

1. ARCHITECTURE
- Prefer static generation (SSG) or hybrid SSR with caching
- Eliminate unnecessary client-side rendering
- Hydrate only interactive components (partial hydration)

2. JAVASCRIPT (CRITICAL)
- Remove ALL unused JS
- Code split at route + component level
- Load non-critical JS ONLY after interaction or idle
- Use:
  requestIdleCallback / dynamic imports
- Move heavy tasks to Web Workers
- Avoid large libraries unless absolutely required

3. IMAGES (BIGGEST WIN)
- Convert ALL images to AVIF + WebP
- Use <picture> with fallbacks
- Implement responsive srcset + sizes
- Preload ONLY the LCP (hero) image
- Ensure exact sizing (no browser resizing)

4. CSS
- Inline critical CSS (above-the-fold)
- Defer all non-critical CSS
- Remove unused styles completely
- Avoid bloated frameworks (or purge fully)

5. FONTS
- Self-host fonts
- Preload critical fonts
- Use font-display: swap or optional
- Match fallback metrics to prevent CLS

6. RENDERING OPTIMIZATION
- Eliminate render-blocking resources
- Use:
  fetchpriority="high"
  rel="preload"
- Reduce DOM size
- Avoid forced reflows

7. LAYOUT STABILITY
- Always define width + height for media
- Reserve space for dynamic elements
- Prevent ALL layout shifts

8. THIRD-PARTY SCRIPTS (STRICT CONTROL)
- Keep only essential scripts
- Load them:
  - after user interaction OR
  - after main content loads
- Replace embeds (YouTube, maps) with lightweight facades

9. ANIMATIONS (NO PERFORMANCE COST)
- Use only:
  transform + opacity
- Avoid layout-triggering properties
- Use GPU acceleration

10. NETWORK & DELIVERY
- Enable Brotli compression
- Use HTTP/2 or HTTP/3
- Deploy via CDN (edge caching required)
- Cache static assets with long TTL (1 year)
- Use immutable caching strategy
