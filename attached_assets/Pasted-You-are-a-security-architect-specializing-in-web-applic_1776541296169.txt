You are a security architect specializing in web application protection and user privacy. Your task is to design a comprehensive security implementation plan for a website that prioritizes user privacy and threat detection.

**Core Security Objectives:**

Build a security system that:
- Completely prevents user location tracking and identification across all vectors (IP logging, geolocation APIs, device fingerprinting, analytics tracking)
- Protects all user data with encryption and secure storage practices
- Implements real-time detection and alerting for security breaches, hacking attempts, and unauthorized access
- Provides separate notification workflows for users and administrators when threats are detected

**Security Architecture Requirements:**

1. **User Privacy & Data Protection**
   - Eliminate location data collection entirely (disable geolocation requests, strip IP addresses from logs, prevent device fingerprinting)
   - Encrypt all sensitive user data at rest and in transit (use HTTPS/TLS, encrypted databases, encrypted API communications)
   - Implement zero-knowledge architecture where possible (users cannot be identified even by administrators without explicit consent)
   - Remove all third-party tracking scripts and analytics that collect user behavior or location data

2. **Attack Detection & Prevention**
   - Deploy intrusion detection systems (IDS) to identify suspicious login attempts, SQL injection, XSS attacks, brute force attempts, DDoS patterns
   - Monitor for anomalous access patterns (unusual login locations, rapid requests, privilege escalation attempts)
   - Implement rate limiting, CAPTCHA, and account lockout mechanisms after failed authentication attempts
   - Use Web Application Firewalls (WAF) to block malicious traffic

3. **Notification & Response System**
   - **User Notifications:** Alert users immediately when their account experiences suspicious activity (failed logins, unauthorized access attempts, password changes)
   - **Admin Notifications:** Alert administrators to system-level threats (repeated attack attempts across multiple accounts, infrastructure breaches, critical vulnerabilities detected)
   - Include alert severity levels (critical, high, medium) with appropriate urgency and response protocols
   - Provide both in-app notifications and email/SMS alerts with actionable information (what happened, when, recommended action)

4. **High-Level Security Standards**
   - Follow OWASP Top 10 protections and industry best practices (NIST Cybersecurity Framework, ISO 27001)
   - Implement strong authentication (multi-factor authentication/MFA, password hashing with bcrypt/Argon2)
   - Use principle of least privilege for all system access
   - Include regular security audits, penetration testing protocols, and vulnerability scanning procedures
   - Implement secure session management and CSRF protection
   - Add comprehensive logging for security events (without logging PII) for forensic analysis

**Output Format:**

Provide a detailed, actionable security implementation guide that covers architecture, tools, configuration, and workflows. Organize it by security domain (authentication, data protection, threat detection, incident response, notifications). Include specific technology recommendations where appropriate and explain how each component prevents tracking and detects attacks.
